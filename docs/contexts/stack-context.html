05 — Code conventions

House style for both sides

Imports

Absolute imports rooted at @/ on both sides (tsconfig paths). Group: external → @/types → @/features → local. No deep imports across feature-pack boundaries.

Files

kebab-case for files (site-config.ts). PascalCase for components (IntakeForm.tsx). One default export per component file.

Target: hooks beside owning components. Fork today: client/src/hooks/useApiAuth.ts — legacy global folder; relocate when touching auth wiring.

Types

interface for extensible object shapes, type for unions. zod at route boundaries (content pack Phase 6.2+); matching z.object() per Mongoose schema as packs gain writes.

Async

async/await throughout; errors at route boundary. Fork today: asyncHandler + errorHandler in server/src/middleware/errorHandler.ts. express-async-errors is optional future adoption — do not assume it is imported.

Tests

Target: Vitest, co-located *.test.ts, integration tests under tests/integration/. Not configured in fork yet.

Comments

Comment why, not what. A reference to the brief that motivated a non-obvious choice (// see template-architecture-brief §03 — single seam for phase 2) is worth more than three lines explaining the code.

06 — Anti-patterns specific to this codebase

Things Claude should refuse

!

Editing core code from a client build. Per the template brief: client builds add Custom*.tsx files and override via site.config.ts / site.theme.ts. They do not edit feature packs, types, the resolver, or middleware.

i

Phase 4 complete (v0.5.0). Four vertical presets, init-vertical, BOUND_TENANT_ID local binding, tenant-aware doctor. Phase 3 payments (Stripe + Square, webhooks) unchanged.

!

Reaching past getSiteConfig(req). (Phase 1+) No route reads tenant context any other way. No process.env.TENANT_ID in route handlers. One seam, one entry.

!

Importing Stripe or Square SDKs outside server/src/providers/. (Phase 3+) Routes call getPaymentProvider().

!

Adding a model without tenantId. (Phase 1+) Every new Mongoose schema declares it, compound-indexed. Single-tenant uses 'default', not absence.

!

Adding a token category outside the registry. Per the visual brief: leaf-level overrides only. A tenant can override color.brand.primary; they cannot introduce color.brand.tertiary. The category shape is platform-owned.

!

Hardcoding a CSS value that has a token. If --color-brand-primary exists, use var(--color-brand-primary). Hardcoded hexes outside theme/ are a build error.
