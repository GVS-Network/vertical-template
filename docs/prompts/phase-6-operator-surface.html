PHASE 6 Operator Surface · v0.6.0 → v0.7.0

Phase 6 — Operator surface.

Ship tenant-facing admin, content writes, intake email, and public Markdown rendering in core — before any first-client engagement. Replaces per-client Storyblok/Getform patterns with GVSN-native APIs and /admin. See docs/amendments/phase-plan-amendment-v1.md.

01 — Prerequisites

Before you start this phase

  • Phase 5 closed, tagged v0.6.0.
  • Amendment v1 accepted — phase map is 0–5 closed, this is 6, first client is 7, multi-tenant is 8.
  • Read docs/phase-6-prompt-6.1-resolutions.md after prompt 6.1 (defaults already locked for Resend, structured event fields, explicit features.admin).
→

Two admin surfaces. /admin/* = tenant owner (this phase). /__admin/* = platform operator (Phase 8). Never merge routes or auth models.

02 — Deliverables

What this phase produces

Content write API

POST/PUT pages and posts; zod validation; auth via writeGuards; public GET returns published only.

Event posts

Structured event fields on Post; public list via ?tag=event.

Intake notifications

providers/notifications/ + Resend; email to siteConfig.contact.email on submission.

Submissions inbox API

Auth-gated list + mark processed for tenant admins.

features.admin pack

/admin UI: pages, events, products, submissions. Toggle-mounted; requires auth.

MarkdownBody

Public PageRenderer + PostDetail render stored Markdown.

03 — Change taxonomy

Type A vs C

Type A
Core platform

New admin pack, notification provider, content writes, Post schema extension, features.admin on SiteConfig — propagates to all builds via pull.

Type C
Per-build later

Terrible Gerald's custom public frontend stays Phase 7. No client-specific code in Phase 6.

05 — Acceptance criteria

Phase 6 is done when…

  • Authenticated user can CRUD pages, event posts, and products via /admin.
  • Public site shows published Markdown; events list from API.
  • Intake form submission sends email to contact.email and appears in admin inbox.
  • features.admin: false mounts zero admin routes.
  • contract:check, npm run build, doctor pass.
  • Tagged v0.7.0.
